Le développement assisté par IA manque de structure

Claude Code est puissant — mais sans cadre, chaque projet repart de zéro.

🔀

Workflows incohérents

Chaque feature, bugfix ou deploy suit un chemin différent. Pas de standard, pas de revue systématique, des oublis fréquents.

🧠

Contexte perdu entre sessions

Claude ne se souvient pas de la session précédente. Version en cours, décisions prises, règles du projet — tout se répète.

🏝️

GitHub en silo

Issues, milestones, PRs et releases sont gérés manuellement, en dehors du flux de développement.

🧊

Template figé après l'init

Les outils d'amorçage copient des fichiers et disparaissent. Impossible de récupérer les améliorations sans tout refaire.

Un cadre complet, toujours à jour

Architecture TEMPLATE / PROJET : les commandes viennent de GitHub, votre code reste le vôtre.

🎯

Workflows orchestrés par le CDP

Le Chef de Projet coordonne planner, devs, reviewer, QA, doc et deploy. Chaque commande suit un flux précis avec points de validation.

💾

Mémoire de session persistante

/start-session lit l'état du projet, la version courante, le milestone actif et le travail en cours. Aucune répétition.

🐙

GitHub natif

Issues, milestones, PRs, releases, CI/CD — tout s'orchestre via context/GITHUB.md partagé entre tous les agents.

🔄

Toujours à jour

Un seul fichier à copier. /init-project fetche la dernière version depuis GitHub et migre automatiquement les projets existants.

Fonctionnalités

🏁

Gestion des Milestones Nouveau

Créez un milestone, associez vos issues, suivez la progression en temps réel. Clôture automatique au deploy prod avec gestion des issues non terminées.

📋

Backlog intégré Nouveau

/backlog liste vos issues GitHub, détecte le type de workflow depuis les labels et propose l'assignation au milestone actif.

🤖

Agents spécialisés

CDP, Planner, Dev, Reviewer, QA, Security, Doc, Deploy, Marketing — chaque agent a un rôle précis et communique via SendMessage.

🚀

Deploy QUALIF → PROD

Pipeline de déploiement en deux étapes avec smoke tests, rollback intégré et clôture du milestone en production.

📣

Site marketing auto-généré Nouveau

Ce site est généré par /marketing depuis les issues du milestone clos, le CHANGELOG et le README. Bilingue FR/EN, pushé sur gh-pages.

🐙

Contexte GitHub centralisé Nouveau

context/GITHUB.md unifie tous les patterns gh CLI : issues, milestones, PRs, releases, CI. Un seul fichier à mettre à jour.

⬆️

Migration v1→v2 automatique Nouveau

Les projets v1 sont détectés et migrés : git rm --cached des fichiers template, .gitignore appliqué, commit propre.

🗺️

Workshop de cadrage

Pour les nouveaux projets : entretien guidé en 6 phases pour générer un CLAUDE.md complet avec vision, stack et conventions.

Installation

Un seul fichier suffit pour démarrer.

1

Copier init-project.md

Téléchargez le seul fichier nécessaire au bootstrap :

# Créer le dossier de commandes
mkdir -p .claude/commands

# Télécharger init-project.md
curl -o .claude/commands/init-project.md \
  https://raw.githubusercontent.com/CCoupel/claude_project_template/main/.claude/commands/init-project.md
2

Lancer /init-project

Ouvrez le projet dans Claude Code et tapez :

/init-project

Claude fetche automatiquement tous les fichiers du template depuis GitHub, détecte votre stack et génère la configuration.

3

Travailler

Votre équipe d'agents est prête :

/start-session /feature /bugfix /backlog /milestone /review /qa /deploy prod /marketing

Projet existant (v1) ?

Copiez init-project.md dans votre projet et lancez /init-project — la migration v1→v2 est automatique.
